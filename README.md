# Career Navigator: AI Skill Gap Analyzer 🚀

A production-grade system that analyzes a user's resume against a Job Description (JD) or a predefined Career Role to generate a targeted, 5-week learning roadmap.

## 🌟 Features
- **Dual-Flow Exploration**: Switch between JD parsing and Role discovery.
- **Skill Profile Analytics**: Real-time match percentage and categorized strengths/gaps.
- **AI-Powered Roadmaps**: Personalized 5-week learning paths generated by Google Gemini 2.5 Flash.
- **Deterministic Matching**: No NLP hallucinations during gap analysis; strict keyword and alias mapping.
- **Responsive UI**: Premium glassmorphism design with Lucide React iconography.

## 🛠️ Tech Stack
- **Frontend**: React, Tailwind CSS, Lucide React.
- **Backend**: Spring Boot 3, Java 17, Spring Data JPA.
- **Database**: PostgreSQL (Relational mapping for Roles/Skills).
- **AI Engine**: Google Gemini 2.5 Flash API.

## 📦 Project Structure
- `/backend`: Spring Boot application and database entities.
- `/frontend`: React application and UI components.
- `/docs`: System design and architectural documentation.
- `/data`: Synthetic dataset for testing and demonstration.

## 🚀 Getting Started

### Prerequisites
- Java 17+
- Node.js 18+
- PostgreSQL
- Gemini API Key ([Get one here](https://aistudio.google.com/))

### Backend Setup
1. Configure `backend/src/main/resources/application.properties` with your DB credentials and API key.
2. Run the application:
   ```bash
   ./mvnw spring-boot:run
   ```

### Frontend Setup
1. Install dependencies:
   ```bash
   npm install
   ```
2. Run the dev server:
   ```bash
   npm run dev
   ```

## 📐 Design Decisions & Tradeoffs

### 1. Deterministic vs. Probabilistic Matching
- **Decision**: Used an internal keyword-alias registry instead of a pure NLP extraction API.
- **Tradeoff**: Required a more robust data seeder but ensured 100% accuracy in identifying "Gaps". Users never get a roadmap for a skill they already listed on their resume under an alias.

### 2. LLM Boundaries
- **Decision**: LLM is restricted only to pedagogical planning (the roadmap).
- **Tradeoff**: More backend code for the matching engine, but lower latency and zero hallucination risk for core gap analysis.

### 3. PostgreSQL Over NoSQL
- **Decision**: Used PostgreSQL for role/skill relationships.
- **Tradeoff**: Slightly more rigid schema, but provides strong consistency for priority-based skill selection (Top K).
